# 🗂️ notes-organizer

> 一个面向 **Markdown / Obsidian 笔记库** 的命令行整理工具：自动重组目录、移动文件，并为相关内容补充双向链接。

## ✨ 项目简介

`notes-organizer` 适合个人知识库、学习笔记和项目文档归档场景。
运行脚本后，你可以通过交互式问答选择分类方式、命名风格、是否创建子目录、是否生成双向链接，以及是否清理空目录。

---

## 🚀 功能亮点

- **交互式整理向导**：无需配置文件，直接通过命令行选择整理方案
- **多种分类策略**：支持按主题、学习阶段、项目、时间或自定义方式分类
- **自动目录重组**：创建新的目录结构，并移动对应的 Markdown 文件
- **知识关联补链**：基于关键词与内容相似度，自动插入 `[[笔记名]]` 风格双向链接
- **整理日志输出**：将执行配置和操作记录保存到 `.organize_log.json`
- **可选清理操作**：支持清理空目录，保持笔记库整洁

---

## 📦 适用场景

- 整理混乱的 Obsidian / Markdown 笔记库
- 归档课程学习笔记、技术文档、项目资料
- 为已有知识点自动补充基础关联链接

---

## 🛠️ 快速开始

### 1) 在 AI Agent中

```
/organize_notes [目录]
```

### 2) 按提示完成配置

程序会依次询问：

| 配置项 | 可选内容 |
| --- | --- |
| 分类方式 | 按主题 / 按学习阶段 / 按项目 / 按时间 / 自定义 |
| 命名风格 | 数字前缀 / 中文 / 英文 / 自定义 |
| 子目录 | 是否创建二级子目录 |
| 双向链接 | 是否添加链接、插入位置、显示格式 |
| 清理策略 | 清理空目录 / 清理空目录和重复文件 / 不清理 |

---

## 🔄 工作流程

1. **扫描** 目标目录中的 `.md` 文件  
2. **创建** 对应的分类目录结构  
3. **移动** 笔记到推断出的目标目录  
4. **分析** 文件内容相似度并补充关联链接  
5. **保存** 操作日志并按需执行清理  

---

## 🔗 双向链接示例

```markdown
# 文件标题

> **相关链接**: [[文件A]] | [[文件B]] | [[文件C]]

正文内容...
```

当选择详细模式时，也会输出类似 `[[文件A|文件A]]` 的写法。

---

## 📝 注意事项

1. **建议先备份原始笔记目录**，再进行批量整理。
2. 当前分类逻辑主要基于 **关键词匹配**，适合初步自动归类。
3. 链接关联基于 **文本相似度分析**，更适合作为辅助整理而非完全替代人工校对。
4. 执行记录会保存在根目录下的 `.organize_log.json` 中，便于回溯整理过程。
